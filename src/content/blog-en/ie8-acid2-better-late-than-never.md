It seems the [Internet Explorer](http://www.microsoft.com/spain/windows/products/winfamily/ie/default.mspx) development team is finally stepping up, and the upcoming version 8 of their browser will at last respect web standards. Dean Hachamovitch, General Manager of the IE development team, announced today on [IEblog](http://blogs.msdn.com/ie/) that [IE8 passes the Acid2 test](http://blogs.msdn.com/ie/archive/2007/12/19/internet-explorer-8-and-acid2-a-milestone.aspx), joining other browsers that already do — [Safari](http://www.apple.com/es/safari/) (the first to pass it), [Opera 9](http://www.opera.com/), [Firefox 3](http://www.mozilla.com/en-US/firefox/all-beta.html), and others.

A rather well-timed announcement, incidentally, given that on 13 December [Opera Software filed a complaint with the European Union](http://www.elpais.com/articulo/internet/Opera/denuncia/Microsoft/dificultar/competencia/elpeputec/20071213elpepunet_7/Tes) partly for this very reason. Microsoft, for their part, claims that standards compliance had been a priority from the start of the project.

If you're not a web developer, all this talk of the [Acid2 test](http://en.wikipedia.org/wiki/Acid2) and web standards probably means nothing to you, so let me explain briefly. The test consists of a drawing constructed using HTML and CSS 2.0 standards — elements and features that every browser is supposed to support. If the browser actually respects those standards, it should display a smiley face; otherwise, you get results [like](http://en.wikipedia.org/wiki/Image:Ieacid2.png) [these](http://en.wikipedia.org/wiki/Image:Ie7acid2.png).

If this announcement holds true in the final release of IE8, it would mean the end of one of the biggest headaches web developers face on every project: **getting Internet Explorer to display a webpage the way it's supposed to**.
